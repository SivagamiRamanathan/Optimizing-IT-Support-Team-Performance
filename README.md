**Section 1 – Title**
**Optimising IT Support Team Performance Using Analytics**     
This project focuses on analyzing IT support team data to evaluate performance, identify inefficiencies, and improve overall service quality.

**Section 2 – Problem Statement**          
IT support teams handle multiple tickets daily, including technical issues, system errors, and user complaints.
However, organizations often face challenges such as:
Delayed ticket resolution
Uneven workload distribution
Low customer satisfaction
Lack of performance tracking
There is a need for a data-driven solution to monitor Key Performance Indicators (KPIs) and improve IT support team efficiency.

**Section 3 – Technical Support Dataset Description**    
The Technical Support Dataset contains service ticket records generated by an IT support team. The data represents real-time operational activities including ticket creation, assignment, resolution, and customer feedback.
This dataset is used to analyze team efficiency, service quality, and performance trends.   

Ticket_ID – A unique identification number assigned to each support ticket. It is used to track and reference individual issues.            
Issue_Category – Describes the type of problem reported, such as hardware issues, software errors, network problems, or login failures.     
Priority_Level – Indicates the urgency of the ticket. It is usually classified as Low, Medium, High, or Critical.          
Ticket_Status – Shows the current stage of the ticket, such as Open, In Progress, Resolved, or Closed.          
Assigned_Engineer – The name or ID of the IT support team member responsible for handling the ticket.          
Department – Specifies the department that raised the support request.      
Created_Date – The date and time when the ticket was initially created.            
First_Response_Time – The time taken by the support team to provide the first response after the ticket was created.          
Resolution_Date – The date and time when the issue was completely resolved.            
Resolution_Time – The total time taken to resolve the issue, calculated from the ticket creation time to the resolution time.         
SLA_Status – Indicates whether the ticket was resolved within the defined Service Level Agreement time. It is usually marked as Met or Breached.      
Customer_Satisfaction_Score – The feedback rating provided by the user after the issue was resolved, typically measured on a scale from 1 to 5.          

**Data Processing Steps:**         

Removed missing or null values        
Standardized date formats           
Cleaned inconsistent category names        
Created calculated columns for Resolution Time            
Verified data accuracy             

**Purpose of Dataset**         

The dataset is used to:         
Measure IT support performance           
Track ticket resolution efficiency             
Monitor SLA compliance                
Analyze workload distribution            
Evaluate customer satisfaction trends                    
This dataset forms the foundation for building KPIs and dashboards in Power BI.   

**Section 4 – Key Performance Indicators (KPIs)**

Key Performance Indicators (KPIs) are measurable metrics used to evaluate the performance and efficiency of the IT support team.

The following KPIs were developed in this project:

Total Tickets – The total number of support tickets created within a specific time period.

Resolved Tickets – The total number of tickets successfully resolved by the support team.

Pending Tickets – The number of tickets that are still open or in progress.

Average Resolution Time – The average time required to resolve support tickets.

First Response Time – The average time taken by the support team to respond to a ticket after it is created.

SLA Compliance Rate – The percentage of tickets resolved within the defined SLA time limit.

Tickets by Priority – Shows how many tickets fall under each priority level.

Tickets per Engineer – Displays how many tickets are handled by each support engineer.

Customer Satisfaction Score (CSAT) – The average feedback rating given by users after issue resolution.

These KPIs help monitor team productivity and improve service performance.
